# Role & Objective
You are an expert React Native & Expo developer. Refactor the "Du lịch" (Tourism) screen to fix layout scrolling behaviors and layout bugs based on UI requirements.

## Current Issues (Based on Screenshot Analysis)
- The header and search container are missing or getting cut off when scrolling.
- When pulling/scrolling to the top (`scroll to top`), the layout must reveal a fixed/sticky Header and a professional Search Bar.
- Component structural hierarchy is overlapping, causing content to bleed into the top status bar area incorrectly.

## Requirements & Layout Specifications

### 1. Root & Navigation Layout
- Use `SafeAreaView` from `react-native-safe-area-context` to handle the notch and status bar properly.
- Wrap the entire content within a container that correctly structures the Header, Search Bar, and Scrollable Content.

### 2. Header & Search Component (Must show at the top)
Create or position the Header and Search Bar so that they are fully visible when the user is at the very top of the page:
- **Header:** Contains the app branding/title (e.g., "BenThanh Tourist" style or section title) and optional utility icons (e.g., Notification or Cart).
- **Search Bar:** An interactive input text box with a search icon placeholder ("Tìm kiếm tour, điểm đến...").
- **Behavior:** This header section can either be `sticky` (stays fixed at the top while content scrolls underneath) or part of the `ListHeaderComponent` of a `FlatList` so it naturally appears when scrolled to the top.

### 3. Main Content Area
- Restructure the main container. If using a `ScrollView`, ensure `nestedScrollEnabled={true}` is handled if there are horizontal lists inside (like the "Tin tức nổi bật" horizontal cards).
- Alternatively, use a parent `FlatList` where the Header, Search Bar, Banner, and Tour Promotion Card are part of the `ListHeaderComponent`, and "Tin tức nổi bật" / Footer are rendered systematically to avoid the *"VirtualisedLists should never be nested"* warning.
